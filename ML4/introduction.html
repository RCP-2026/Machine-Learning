    Introduction ML4

    High-dimensional datasets are common in biological and medical research, where measurements may span many correlated variables, features or spatial dimensions. As a type of unsupervised transformation, dimensionality reduction provides a principled way to simplify datasets like this, while preserving their dominant structure and variability. By reducing redundancy and focusing attention on the most informative directions in the data, dimensionality reduction supports more efficient computation, clearer visualisation and more robust downstream analysis.

    This lesson introduces dimensionality reduction through Principal Component Analysis (PCA), first by building intuition around variance and projection, and then by applying PCA to real datasets. PCA is framed as a re-expression of data in terms of orthogonal directions that capture maximal variance. This provides a compact representation of high-dimensional data and a quantitative basis for selecting an appropriate number of features.

    Variance as Information Builds on basic statistics and variance calculations np.var(X, axis=0) Use variance as a measure of information content along a direction Forgetting to centre data before variance-based analysis

    PCA as a Projection Combines variance and projection concepts PCA().fit(X) Interpret PCA geometrically as projection onto orthogonal directions of maximal variance Treating PCA as a black-box transformation
    Principal Components Builds on orthogonality and linear algebra intuition pca.components_ Identify directions that capture maximal variance in the data Confusing components with original features
    Explained Variance Extends variance concepts to transformed space pca.explained_variance_ratio_ Quantify how much information each component captures Misinterpreting cumulative variance
    Dimensionality Selection Builds on explained variance analysis np.cumsum(...) Choose an appropriate number of components to retain Relying on arbitrary cutoffs without justification
    PCA on Real Data Applies PCA to biological-style datasets pca.transform(X) Reduce dimensionality for visualisation and exploratory analysis Forgetting to apply the same transformation to new data
    Limitations of PCA Conceptual extension of linear methods — Recognise when PCA may fail due to non-linearity or scaling issues Applying PCA without standardisation
